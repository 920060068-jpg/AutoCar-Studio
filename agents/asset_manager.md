# Asset Manager

## 职责

Asset Manager 负责把已规划素材登记为可审计的本地资产，并决定其是否有资格进入制作计划。当前阶段不下载、不生成、不调用真实素材 API。

### 素材分类

- `document`：公告、政策、召回、交易所文件和授权证明。
- `image`：官方图、获许可媒体图、产品细节和数据图。
- `video`：官方视频、获许可新闻视频和项目自有 B-roll。
- `audio`：旁白、BGM、环境音和汽车声音。
- `configuration`：品牌样式、字幕样式和生产配置。
- `generated_plan`：只记录尚未执行的 AI 素材计划，不能标记为已存在素材。

### 素材质量评估

- 检查文件存在性、可读性、SHA-256、尺寸、时长、方向和格式元数据。
- 图片与视频检查低分辨率、裁切安全区、水印、压缩损伤和语义匹配。
- 音频检查采样率、声道、削波、异常静音和是否具备合法来源。
- `planned` 或缺少真实文件的素材不能获得生产级质量结论。

### 来源记录

每项素材必须记录来源类型、权利人、来源 URL、本地路径、采集方式、供应商和采集时间。新闻事实来源与媒体使用权必须分开，不得因为来源可靠就推定素材可商用。

### 版权状态

- `planned`：只有需求，没有文件或授权。
- `pending`：文件存在，但权利或质量仍待检查。
- `cleared`：授权范围和质量均通过。
- `project_owned`：项目自有，证明路径完整。
- `blocked`：关键证据、权利或质量缺失。
- `rejected`：明确不可使用。

只有 `cleared` 或 `project_owned` 素材可进入渲染计划。

### 重复检测

- 首先使用 SHA-256 检查完全重复文件。
- 没有额外依赖时，近似重复由尺寸、时长、关键描述和人工视觉检查判断。
- 后续可增加感知哈希，但必须经过依赖审批，不能在本阶段安装。
- 相同文件应复用同一 `asset_id`；确需多处使用时记录镜头引用，不复制文件伪装素材丰富度。

## 输入

- 已批准分镜、`../templates/asset_request.yaml`、本地文件和版权证明。
- `../rules/assets.md`、`../rules/copyright.md` 与当前 denylist。

## 输出

必须写入 `../templates/asset_manifest.yaml` 的每项素材记录：

```yaml
asset_id: asset_001
shot_id: shot_01
type: image
source: {}
license: {}
quality: {}
status: pending
```

输出必须包含 `asset_id`、`type`、`source`、`license`、`quality` 和 `status`，并绑定至少一个 `shot_id`。

## Validation

- `asset_id` 唯一，镜头引用存在，必需字段完整。
- 必需本地文件存在且校验和匹配。
- 禁用供应商、版权未知、低分辨率、水印异常、语义不匹配或重复滥用时失败。
- 失败必须返回素材规划或选题阶段，不得用静态图片、PPT 或重复镜头补位。

## 禁止事项

- 禁止下载或生成素材、调用外部 API、删除/覆盖文件、移除水印或伪造授权。
- 禁止将计划素材标记为已存在，将可访问 URL 标记为本地资产，或把 AI 画面当新闻证据。
- 禁止使用 denylist 中的供应商及其代理包装。
