# Remotion Engine 规则

本文件定义主时间线和输出的引擎级契约，补充 `remotion.md`。当前阶段只设计架构，不调用 Remotion。

## 时间线管理

- Remotion 是唯一主时间线；所有 Scene、Shot、Audio 和 Caption 必须转换为整数帧。
- 时间换算统一使用 `frame = round(seconds × fps)`，总帧数必须与批准时长一致；任何舍入差异在最后一个镜头显式处理。
- `VideoProject` 锁定 Storyboard、Asset Manifest、Motion Plan、Audio Plan、Caption 和 RenderConfig 的 revision 与校验和。
- Scene 组织叙事章节，Shot 是最小时间单元；不得用浏览器真实时间、网络返回时间或非确定性状态控制画面。
- 镜头不得负时长、重叠或产生未说明空帧；转场占用必须在相邻 Shot 的帧预算中明确。

## 素材组合

- 只允许使用 Asset Manifest 中状态为 `cleared` 或 `project_owned` 的固定本地素材。
- 每个媒体引用必须绑定 `asset_id`、`shot_id`、SHA-256、许可状态和预期使用范围。
- Motion Components 作为不可变视觉层接入，不能在主时间线中静默修改其数据或首帧。
- 第 0 帧必须与批准预览首帧一致；代表帧检查通过前不得执行全片渲染。
- 素材缺失、低分辨率、重复滥用、版权未知或语义不匹配时停止，不得用占位图生成正式输出。

## 字幕同步

- Caption 时间以最终 MiniMax 旁白为基准，并换算为帧区间。
- Caption 文案必须来自已批准 Audio-Fit Script；没有 `audio_preflight_pass: true` 的项目不得进入正式旁白合成。
- 字幕内容必须与批准口播一致；数字、单位、车型和品牌名称不能在渲染层改写。
- 旁白留白区间不得生成虚假口播字幕；非口播信息标签必须使用独立层级，不得伪装成正在朗读的 Caption。
- 字幕不得重叠、超出总时长或越过安全区；镜头切换时不得无意截断语义。
- 关键词和数字强调必须遵守 `subtitle_engine.md`，并保留普通可读文本作为语义主体。
- 旁白或剪辑时长变化后必须重新生成全部字幕时间，禁止只移动单个错误字幕掩盖同步问题。

## 音频混合

- 主旁白供应商为 MiniMax，默认声音为沉曜男声；Remotion 只组合已批准音频，不生成或替换声音。
- Remotion 只能组合 Measured Audio Fit PASS 的旁白轨；不得通过改变视觉时序或机械压缩音频来掩盖旁白过长。
- 旁白轨可以包含已规划的自然留白；整条音频轨与视频等长不代表语音必须全程占用。
- 项目音频母版按 48 kHz 规划；旁白、BGM、环境音和车辆音效使用独立轨道。
- BGM 在旁白下按 Audio Plan 执行 ducking；所有增益、淡入淡出和声道布局必须可重放。
- 检查削波、未声明的异常静音、相位、声道、真峰值、响度和音画同步；已规划的自然留白不得误报。
- 车辆声音必须对应实际车型和动力类型；设计音效必须有标签和版权记录。

## 输出规范

- 默认分辨率：1080 × 1920；画幅：9:16；帧率：30 fps；时长：38–45 秒。
- RenderConfig 必须显式指定视频 codec、音频 codec、采样率、像素格式、色彩空间和输出路径。
- 默认交付规划为 MP4 容器、H.264 视频、AAC 音频；正式编码参数需在依赖获准后通过样片校准。
- 每次渲染必须输出日志、代表帧、字幕文件、配置摘要和最终 SHA-256。
- 输出文件只能进入 QC，不得自动发布；自动审核版本必须与最终校验和一致。

## 禁止事项

- 禁止安装或调用 Remotion、生成视频、连接外部 API 或发布平台。
- 禁止 CSS/真实时钟驱动的非确定动画、动态远程 URL 和运行时素材替换。
- 禁止跳过预览首帧、素材版权、字幕、音频或技术 QC。
- 禁止把规划状态、代表帧或渲染启动写成成片通过。
