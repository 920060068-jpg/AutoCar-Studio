# HyperFrames Engine 规则

本文件定义 Motion Component 的引擎级契约，补充 `hyperframes.md` 的职责边界。当前阶段只设计规则，不调用 HyperFrames。

## 镜头运动规则

- 每个 `shot_id` 只能有一个主要运动意图；次级运动不得与主体竞争。
- 运动必须来自批准的 `motion_plan.yaml`，使用确定性时间和帧数。
- 优先使用位置、缩放、旋转和透明度等可预测属性；禁止依赖真实时钟、随机数或运行时网络状态。
- 推、拉、摇、移、跟踪和环绕必须符合镜头焦段、主体方向和前后镜头连续性。
- 预览首帧必须对应 Motion Component 第 0 帧；首帧变化必须新建 revision。
- 运动不能扭曲车身比例、改变数据含义或伪造车辆性能。

## 视觉包装规则

- 必须绑定一个已批准 `brand_styles/*.yaml`，但品牌参考不得覆盖项目可读性和安全区。
- 图层顺序为：证据/主体 → 解释信息 → 字幕预留 → 装饰；装饰不得遮挡证据和来源。
- 一屏只保留一个视觉中心，标题、数据和标注必须有明确层级。
- 屏幕文字必须来自批准脚本或事实数据，禁止在动画阶段改写数字和结论。
- 禁止用模板装饰代替真实素材、视觉语义或镜头变化。

## 数据动画规则

- 每个数字、比例和排名必须绑定 `claim_id`、单位、统计口径和来源。
- 数字动画的终值必须精确等于已核验值；不得通过不合理起点、截断坐标或速度夸大差异。
- 比较动画必须统一时间、地区、版本、单位和比例尺。
- 先显示指标名称与口径，再揭示数值和比较结论。
- 缺失值、估算值和预测值必须显式标记，不能与已确认数据使用相同状态。

## HUD 规则

- HUD 只用于技术、传感器、驾驶辅助或数据解释场景，不能成为默认“科技感”装饰。
- HUD 元素必须对应可解释的信息，不得伪造车辆真实 UI、传感器识别或驾驶状态。
- 保持主体、字幕和平台 UI 安全区域；边缘标线不得贴近裁切区。
- 同时活动的 HUD 模块应限制在必要数量，避免持续闪烁、扫描和无意义数字滚动。
- 模拟界面必须标记为示意，真实车机界面必须使用已授权素材。

## 转场规则

- 转场服务叙事关系：延续使用淡化或匹配切换，信息转折使用受控遮罩，章节变化可使用短暂结构性过渡。
- 转场必须记录 `from_shot_id`、`to_shot_id`、类型、持续帧和负责层。
- 默认转场保持简短；不得遮住关键数字、来源或字幕结尾。
- 连续镜头的主体方向、颜色和空间关系必须在转场前后可理解。
- Motion Component 只能提供局部转场资产；最终跨镜时机由 Remotion 主时间线决定。

## 禁止事项

- 禁止拥有主时间线、混合音频、生成旁白、输出最终成片或执行发布。
- 禁止调用 TTS、外部 API、未批准插件或 denylist 供应商。
- 禁止随机动画、无限循环、异步时间线、CSS 自动播放时间和不可重放行为。
- 禁止 PPT 翻页、纯图片轮播、一句话一张图、无目的缩放和高频闪烁。
- 禁止把生成示意当新闻证据，或用 HUD、数据动画掩盖事实不确定性。
