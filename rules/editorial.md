# 编辑规则

## 目标

把已核验事实转换为清楚、克制、适合 38–45 秒竖屏视频的汽车新闻文案。

## 强制要求

- 只能使用 `rules/fact_checking.md` 已标记为 `verified` 的事实。
- 每条文案必须有单一中心问题，结构为：钩子 → 核心事实 → 影响 → 明确结论或待观察点。
- 数字、日期、车型、公司名称必须保留对应 `claim_id`。
- 区分“官方确认”“媒体报道”“行业推测”和“编辑判断”，不得混写。
- 标题不得夸大正文，不能用无法证明的“史上第一”“彻底失败”“必然”等表达。
- 报价、补贴、销量必须说明适用时间、地区和统计口径。
- 事故和伤亡内容不使用猎奇、娱乐化或煽动性措辞。
- 选题评分必须考虑可获得素材是否足以支撑动态叙事；素材不足时标记为 `deprioritized` 或 `blocked`，不得用静态图片硬凑成片。

## 输出要求

- 前期方向先使用 `templates/script_structure.yaml`；自动审核批准或显式人工覆盖后映射到 `templates/script.yaml`。
- 标明口播、屏幕文字、发音提示、引用和预计时长。
- 超过配置时长时必须重写，不能通过异常加速语音硬塞。

## 失败条件

无证据事实、标题与正文不一致、时长失控或高风险内容缺少人工标记时，编辑阶段失败。
