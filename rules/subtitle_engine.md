# Subtitle Engine 规则

本文件定义字幕渲染层级和动画约束，补充 `subtitles.md` 的内容与时间规则。当前阶段不生成或渲染字幕。

## 字幕分层

从低到高按以下逻辑管理，具体层级值由未来实现统一定义：

1. `caption_background`：必要的底板、阴影或描边，只为可读性服务。
2. `primary_caption`：与旁白一致的主字幕，始终是语义主体。
3. `keyword_emphasis`：对一个关键词或短语的有限强调。
4. `numeric_emphasis`：数字、单位和口径的组合强调。
5. `source_or_correction`：来源、限制条件和更正信息，不能被其他层遮挡。

字幕层必须与标题、HUD 和平台 UI 协调，不能互相覆盖。

## 关键词强调

- 每条字幕最多强调一个核心词组，不能逐字跳动或全部高亮。
- 强调词必须来自原字幕，不得通过替换词改变事实语气。
- 颜色、字重、下划线或短标记只能选择一个主要手段。
- 高风险新闻避免红色闪烁、警报式描边和情绪化放大。

## 数字动画

- 数字与单位、正负号、百分号和口径必须作为一个语义组。
- 动画终值严格等于已核验数字，保留批准的小数位和格式。
- 禁止从误导性起点滚动、隐藏负号、延迟显示单位或拆开统计口径。
- 时间、价格、销量、续航和比例动画必须绑定 `claim_id`。
- 数值变化只用于表达真实对比；静态事实不需要强制计数动画。

## 安全区域

- 默认竖屏安全区读取品牌样式：顶部至少 150 px、右侧至少 150 px、底部至少 260 px、左侧至少 90 px。
- 主字幕默认不超过两行，按语义断句，不拆开车型、数字和单位。
- 避开人物嘴部、关键车辆细节、来源文字和右侧平台操作区。
- 字幕安全区必须在代表帧和实际视频帧上复核；仅通过坐标计算不能视为最终通过。

## 动画限制

- 入场、强调和退场均由帧驱动，时长短且可读；禁止依赖 CSS 实时时间。
- 默认使用淡入、短距离位移或遮罩揭示，避免弹跳、旋转、逐字飞入和高频闪烁。
- 没有词级时间戳时禁止伪造卡拉 OK 逐字高亮。
- 动画不得造成文字抖动、重影、裁切或与旁白错位。
- 同屏标题、HUD 和字幕同时运动时，只保留一个主要视觉动作。

## 禁止事项

- 禁止改写事实、自动纠错车型名、删减限制条件或把标题当字幕重复铺满画面。
- 禁止字幕越界、超过视频时长、相互重叠或遮挡证据。
- 禁止把字幕做成 PPT 页、逐句独立图片或一句话一张图。
